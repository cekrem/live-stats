# live-stats

> **Note:** This README was generated by Claude. The code was written by [cekrem](https://cekrem.github.io).

Real-time visitor counter for blog posts and web pages, built with [Lamdera](https://lamdera.com) and [Elm](https://elm-lang.org).

## What it does

Embed live-stats as an `<iframe>` on any page and it will show how many people are reading that page right now. A dashboard view (no query param) shows live counts across all tracked pages at once.

- `https://livestats.lamdera.app` — dashboard with per-path visitor counts
- `https://livestats.lamdera.app?some-slug` — single counter for that slug, rendered as a link

The counter updates in real time: the backend broadcasts fresh stats every 5 seconds and prunes sessions that haven't sent a heartbeat in the last 20 seconds.

## How it works

Each frontend pings the backend with a heartbeat every 15 seconds, keyed by the URL query param (the "slug"). The backend tracks live sessions per slug using a simple `Dict ClientId Entry`, computes per-slug visitor counts, and broadcasts the result to all connected clients.

No persistent storage, no database — everything lives in memory on the Lamdera backend.

## Stack

- **[Lamdera](https://lamdera.com)** — full-stack Elm platform (handles websockets, deployment, persistence if needed)
- **Elm 0.19.1** — frontend and backend both written in Elm

## Development

```bash
lamdera live
```

## Deployment

Deployed on Lamdera at [livestats.lamdera.app](https://livestats.lamdera.app).

## Usage (embedding)

Add an iframe to any page you want to track — in this case, posts on [cekrem.github.io](https://cekrem.github.io) — using a unique slug as the query param:

```html
<iframe
  src="https://livestats.lamdera.app?your-post-slug-injected-somehow"
  style="border: none; height: 1.5em; width: 100%;"
  scrolling="no"
></iframe>
```

The iframe renders a plain text link: `Number of people reading this post right now: N`.
